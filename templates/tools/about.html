About This Service

You can use these online tools to clean up HTML produced by such tools as Microsoft Word, Libre Office, Calibre and various Content Management Systems.

This service is optimized for producing HTML for SuttaCentral, but can be used by anyone for any purpose, if you want more options, consider using tidy directly http://w3c.github.io/tidy-html5/. If you are a developer check github where the python scripts which are used under the hood are available (under sc/tools).

Format required for SuttaCentral

If you are working on texts for SuttaCentral, or would like to submit some texts, please follow this process:

  1. Get HTML files of the source.

    This may involve crawling a website with a tool like HTTrack, or converting a .doc or .odt file into HTML in Microsoft Word or LibreOffice. Please make sure you have permission.

  2. Perform preliminary cleanup

    Put the files in a zip and upload to the Cleanup tool to give them a preliminary sanitization and normalization. Using the correct settings with the Cleanup tool can save a lot of manual work especially when the HTML files are generated by a word processor or Content Management system.

  3. Required information

    We need to have the following information for every sutta:

    • The Sutta Unique ID (uid), this forms the URL on SuttaCentral. Digha Nikaya 1, becomes legacy.suttacentral.net/dn1/pi, dn1 is the uid. sn56.11 is the uid for the dhammacakkapavattana sutta.

      The preferred way to specify the sutta uid is by naming the file the uid, for example dn1.html, this will keep things very clear. Another way, which permits submitting multiple suttas per file, is to wrap each sutta in a <section id="uid"> tag.

    • The language. On sutta central we use a Language Unique ID for each language. In most cases, the language uid will be identical to the language ISO code, for example 'en' for english, 'de' for german.

      The preferred way to specify the language uid, is by creating a folder named the language uid, and putting all the files of that language in that folder. Zip the whole lot up and submit it.

    • Metadata is an absolute requirement.

      See some existing texts for examples of metadata. However in brief metadata should contain:

      • The translator and/or editor (Root texts, such as Pali, do not have translators), use an <author> tag
      • The edition or website the text is from, use a <cite> tag.
      • The license the text is made available under, if applicable.
      • The above are not hard and fast requirements, however there must be some kind of metadata giving some kind of attribution

      Metadata may be included in a <div id="metaarea"> tag. If the same metadata applies to many files, you may put the metadata in a file named meta.html and include that file in the same folder as the files it applies to.

      For example a submission of German texts, may involve zipping up a structure like this:

      • de/dn1.html
      • de/dn2.html
      • de/dn16.html
      • de/meta.html

  4. Presentation of content.

    • Every sutta should be headed by an <h1> heading. This may be preceded by a division or subdivision subheading.
    • If there are subsections in a sutta, these should be headed by <h2> through <h6> headings, when there are subheadings, a menu will be generated automatically.
    • All paragraphs should use a <p> tag.
    • Verses should be contained within a <blockquote class="gatha">, each set of verses (i.e. four lines) should be in a <p> tag, the lines should be separated by a <br> tag. If there are multiple sets of verses in a row, all should be contained in the same <blockquote>.
    • If the sutta starts with "Evaṁ me sutaṁ" (or a translation thereof), this should be wrapped in a <span class="evam">
    • The above will generally suffice to create a well-presented text. There are many, many details of possible markup. Please inspect similar existing texts for classes which can be used. You can get clean copies of the texts from legacy.suttacentral.net/downloads, or examine the texts on the site itself, using your browsers right click menu 'inspect element' tool.

  5. Submitting texts to SuttaCentral

    Once the texts have been edited (or even before) you can try uploading them to the Finalize tool. This will report any errors (serious problems which cannot be resolved automatically), and also give information on how it interepts information, for example it will say what sutta and what language it thinks each file refers to. The Finalize tool will take care of many finnicky details of markup and is quite tolerant of the input not being exactly right. Presently, you need to e-mail the zip to a SuttaCentral team member, unless you have direct access to the github data repository.
